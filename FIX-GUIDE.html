🏘️ SocietyPro ERP — Fix Guide

ERR_CONNECTION_REFUSED fix · Server: SERP · ADMIN\SQLEXPRESS

Root Cause: The error ERR_CONNECTION_REFUSED on the login page means your React frontend cannot reach the backend API at http://localhost:5000. Either the backend hasn't been started yet, or SQL Server failed to connect and crashed it.
1
Verify SQL Server is Running
Press Win+R → type services.msc → find these services
  • Find SQL Server (SQLEXPRESS) → must be Running
  • Find SQL Server Browser → must be Running (right-click → Start)
If SQL Server Browser is disabled: right-click → Properties → Startup type → Automatic → then Start
2
Create the Database in SSMS
Open SSMS → connect to ADMIN\SQLEXPRESS with Windows Auth

Open setup.sql from the downloaded zip (backend/sql/setup.sql) and press F5. Or run this quick version:

CREATE DATABASE SocietyERP; GO
After running setup.sql, you should see 8 tables in SocietyERP database.
3
Install & Start Backend MOST IMPORTANT
Open Command Prompt or PowerShell in the backend folder

Navigate to your backend folder:

cd C:\path\to\society-erp\backend

Run these commands ONE BY ONE:

npm install
npx prisma generate
node src/prisma/seed.js
npm run dev
✅ You should see: "SocietyPro ERP Backend RUNNING on port 5000"
Keep this terminal OPEN while using the app. Do not close it!
4
Start Frontend (separate terminal)
Open a NEW Command Prompt window for the frontend
cd C:\path\to\society-erp\frontend
npm install
npm start
Browser opens at http://localhost:3000 automatically.
5
Still Failing? Try SQL Auth Instead
Edit backend/.env and switch to SQL Server Authentication

In SSMS, enable SA login:

ALTER LOGIN sa ENABLE; ALTER LOGIN sa WITH PASSWORD = 'Society@2025'; GO

Then update backend/.env:

DATABASE_URL="sqlserver://ADMIN\SQLEXPRESS;database=SocietyERP;user=sa;password=Society@2025;encrypt=true;trustServerCertificate=true"
Also enable SQL Server Auth in SSMS: Right-click server → Properties → Security → SQL Server and Windows Authentication mode
